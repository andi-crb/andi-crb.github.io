Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

I'd think of them like a planned and regulated town. The HTML is the content of the town, eg "a row of 10 houses there, a primary school here, a main road over there". The CSS is the nature and appearance of these - eg "all houses are two storey, the parks are planted with native grasses, all stop signs are red". The DOM (Document Object Model) is like the town plans. JavaScript can be used to manipulate html content - these can be seen as town edicts directed at all of a particular type of content. For example, "convert all high schools into emergency shelters" or "rename every street after a budgie owned by the recently deceased president".

What is meant by boxifying design?

Boxifying design means concieving of each element of a design (such as a title, or an image) as a box, irrespective of whether it appears to take a box shape. This can involve mentally (or physically) drawing a box around each element.

What is the box model?

The box model is, as described above, a model which sees each element as a box. That box incorporated the content itself, the element (such as an image), padding around the image, then (working outwards) a border and then finally a margin.
